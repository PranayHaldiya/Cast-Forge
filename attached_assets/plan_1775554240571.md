## What is CastForge?

**One-liner:** Type any topic → get a fully produced, multi-voice podcast episode with custom AI hosts, sound effects, and music in 60 seconds.

**Elevator pitch:** CastForge is NotebookLM Audio Overviews, but open and customizable. Instead of two fixed Google hosts discussing your documents, CastForge lets you pick any topic, choose a show format (comedy, debate, explainer, true crime, hot takes), describe the hosts you want ("a sarcastic British woman and an overly enthusiastic Texan man"), and generates a complete produced audio episode — with banter, emotional delivery, sound effects, and a music intro/outro.

**Why it matters:** NotebookLM proved massive demand for AI-generated podcast content (millions of audio overviews generated). But it's locked in Google's ecosystem, offers zero customization, always uses the same two voices, has no sound design, and can't produce shareable clips. CastForge democratizes podcast creation.

## ElevenLabs APIs Used (5 total)

### 1. Voice Design API (Text-to-Voice)
**Purpose:** Generate unique host voices from natural language descriptions.
**Endpoint:** `POST /v1/text-to-voice/create-previews`
**How we use it:**
- User describes each host (e.g., "A deep-voiced, sarcastic British man in his 40s" or "An energetic young woman with a slight Southern accent")
- We generate 3 previews per host using `eleven_multilingual_ttv_v2` model
- Auto-select the best preview and save to voice library via `POST /v1/text-to-voice/create`
- We also provide preset host pairs for quick-start (e.g., "Comedy Duo", "Debate Rivals", "True Crime Narrators")

**Parameters:**
```json
{
  "voice_description": "User's natural language description, 20-1000 chars",
  "text": "Auto-generated sample matching show format",
  "model_id": "eleven_multilingual_ttv_v2",
  "guidance_scale": 5,
  "quality": 0.9,
  "loudness": 0.5
}
```

### 2. Text-to-Dialogue API (eleven_v3)
**Purpose:** Generate the full multi-speaker podcast episode with emotional delivery.
**Endpoint:** `POST /v1/text-to-dialogue`
**How we use it:**
- LLM generates a script as an array of `{text, voice_id}` pairs
- Script includes emotion tags: `[laughing]`, `[whispering]`, `[excited]`, `[sarcastic]`
- Script includes natural interruptions with `—` dashes
- Script includes trailing thoughts with `...`
- Dialogue rendered as a single multi-speaker audio file

**Script format example:**
```json
{
  "inputs": [
    { "voice_id": "host_a_id", "text": "[laughing] Okay, so you're telling me cats knock things off tables on PURPOSE?" },
    { "voice_id": "host_b_id", "text": "Dead serious. There's actual research on this — cats have been observed watching the object fall with what researchers describe as... [whispering] satisfaction." },
    { "voice_id": "host_a_id", "text": "Satisfaction! That's — that's genuinely terrifying." }
  ]
}
```

### 3. Sound Effects API
**Purpose:** Generate contextual sound effects — transitions, stings, ambient beds.
**Endpoint:** `POST /v1/sound-generation`
**How we use it:**
- Generate show-specific SFX based on format:
  - Comedy: rimshot, laugh track, record scratch
  - True Crime: suspenseful drone, door creak, heartbeat
  - Debate: bell ding, crowd murmur, gavel
  - Explainer: gentle whoosh transition, notification chime
- Generated once per format and cached for reuse

**Parameters:**
```json
{
  "text": "dramatic suspenseful orchestral sting for a true crime podcast transition",
  "duration_seconds": 3,
  "prompt_influence": 0.7
}
```

### 4. Music Generation API
**Purpose:** Generate intro/outro jingle matched to the show's vibe.
**Endpoint:** `POST /v1/music/stream`
**How we use it:**
- Generate a 10-15 second instrumental jingle based on show format
- Comedy → upbeat, playful
- True Crime → dark, ambient, piano
- Debate → dramatic, brass
- Hot Takes → punchy, electronic

**Parameters:**
```json
{
  "prompt": "Upbeat playful podcast intro jingle, energetic, short, no vocals",
  "music_length_ms": 12000,
  "output_format": "mp3_44100_128"
}
```

### 5. Text-to-Speech (Flash v2.5) — for quick preview mode
**Purpose:** Fast preview of a single line before committing to full generation.
**Endpoint:** `POST /v1/text-to-speech/{voice_id}/stream`
**Model:** `eleven_flash_v2_5` (~75ms latency)
**How we use it:**
- After Voice Design creates a host, we play a quick sample line in that voice
- User confirms they like the voice before full episode generation

## App Architecture

```
User Input (topic + format + host descriptions)
    │
    ├──→ [1] Voice Design API → Generate unique host voices (parallel)
    │
    ├──→ [2] LLM (GPT-4o) → Generate podcast script with emotion tags
    │
    ├──→ [3] Sound Effects API → Generate format-specific SFX (cached)
    │
    ├──→ [4] Music API → Generate intro/outro jingle (cached per format)
    │
    └──→ [5] Text-to-Dialogue API → Render script with designed voices
              │
              └──→ [6] Audio Mixer → Combine dialogue + SFX + music
                        │
                        └──→ Final episode audio + shareable player
```

Steps 1, 3, and 4 can run in parallel.
Step 2 can start immediately (doesn't depend on voice IDs yet).
Step 5 depends on both step 1 (voice IDs) and step 2 (script).
Step 6 combines everything.

## LLM Script Generation Prompt

The LLM prompt is critical to quality. Here's the system prompt structure:

```
You are a podcast script writer for CastForge. Generate a natural, engaging
podcast script for two (or three) hosts discussing the given topic.

FORMAT: {comedy|debate|explainer|true_crime|hot_takes|interview}

HOSTS:
- Host A: {name} — {description}
- Host B: {name} — {description}

RULES:
1. Write 15-25 exchanges (aim for 2-3 minutes of audio)
2. Make it sound like a REAL conversation — interruptions, reactions, tangents
3. Use emotion tags in square brackets: [laughing], [whispering], [excited],
   [serious], [sarcastic], [shocked], [thoughtful]
4. Use — for interruptions: "I think that's—"
5. Use ... for trailing off: "Well, the thing is..."
6. Include at least 2-3 genuinely interesting facts or insights about the topic
7. Match the FORMAT's tone:
   - comedy: witty banter, unexpected analogies, playful disagreements
   - debate: structured arguments, counterpoints, "devil's advocate" moments
   - explainer: curiosity-driven, "wait, really?" moments, building understanding
   - true_crime: suspenseful pacing, cliffhangers between exchanges, eerie details
   - hot_takes: bold opinions, rapid-fire, "I said what I said" energy
   - interview: one host asks probing questions, the other shares expertise
8. Open with a hook, not "welcome to the show"
9. End with a memorable closing line, not a generic sign-off

OUTPUT FORMAT:
Return a JSON array of objects: [{ "voice": "A" or "B", "text": "..." }, ...]
```

## Audio Mixing Strategy

After generating all audio assets, we mix them:

1. **Intro jingle** (music) — 8-12 seconds, fade in
2. **Cross-fade** into first dialogue line
3. **Dialogue** — the main Text-to-Dialogue output
4. **SFX insertions** — at natural break points (every 5-7 exchanges):
   - Transition whoosh between segments
   - Format-specific stings (rimshot for comedy, suspense drone for true crime)
5. **Outro** — last dialogue line cross-fades into outro jingle, fade out

Mixing approach:
- If FFmpeg is available on Replit: use it for professional audio mixing
- Fallback: use Web Audio API on the client side to layer and mix
- Final output: single MP3 file, 128kbps, 44.1kHz
